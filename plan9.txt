PLAN 9 / 9front QUICK REFERENCE (this machine; NOT Linux). All items below verified by running.

Shell is rc, not bash:
- Pipes: a | b.  Logical: cmd && cmd, cmd || cmd.
- Command substitution is `{cmd} (NOT $(...)). Verified: `{echo hi | wc -c} -> 3.
- Variables are lists: x=(a b); $x.  Assign: x=val.
- Redirect: >file  >>file  <file.  Stderr: >[2]file (NOT bash's 2>file).  Merge stderr into stdout: >[2=1].
- No <(...) process substitution, no <<< here-strings. rc DOES have here-docs (see below).
- Conditionals: if(test -f x){...}; if not {...}.  Loops: for(i in a b c){...}.

C toolchain (amd64) -- VERIFIED:
- Headers: #include <u.h> then #include <libc.h> (that order). No stdio.h.
- Print with print("fmt",...); no printf/puts.  Program ends with exits(nil).
  void main(int argc, char *argv[]){ ...; exits(nil); }
- 6c does NOT take -o. It writes stem.6 into the CURRENT directory:
      6c prog.c            # -> prog.6 in cwd (verified: hello.c -> hello.6 in /tmp)
      6l -o /tmp/prog /tmp/prog.6   # only 6l takes -o; makes the runnable
  Run: /tmp/prog (verified prints correctly). GRAPHICAL: window /tmp/prog.
  If link says "cannot open file: ...6", find the object: ls *.6 (do NOT add -o to 6c).

Command differences an agent gets wrong (VERIFIED):
- grep uses egrep regex but has NO -E flag: -E gives a usage error.
  Write alternation directly: grep 'web|xyz'  (verified: matches "web http ftp").
- ls has NO -a and NO -R (recursive) flag -- both give a usage error. Accepts: -dlmnpqrstuFQT.  lc = columnar.
- NO find, NO head/tail, NO od. First N lines: sed Nq FILE (not head). Line range: sed -n 'A,Bp' FILE.
- NO which. To test whether a command exists: test -f /bin/NAME, or ls /bin/NAME, or whatis NAME.
  When an ls listing is truncated, do NOT conclude from the visible slice -- grep for the name:
  ls /bin | grep '^NAME'  (e.g. hoc sorts under h and is missed if you only see the top of the list).
- To recurse a directory tree, use du -a DIR: it prints "SIZE(KB) PATH" for EVERY file AND dir under it.
  There is NO -type filter, and directories aggregate their contents so they sort ABOVE individual files --
  to find the largest FILE, look past the directory rows yourself. Search names: du -a DIR | grep NAME.
- sed s/a/b/g substitutes; sort / sort -n / sort -r; uniq / uniq -c; wc / wc -l.
- date takes NO +FORMAT (Unix-only). Plain date prints "Wed Jul 4 HH:MM:SS TZ YYYY"; pull fields
  with awk, e.g. the time is `{date | awk '{print $4}'}. Seconds since epoch: date -n.

Useful files/commands:
- Calculator: hoc is the standard interactive calculator (C-like floating-point expressions);
  dc (RPN) also exists. abaco and mothra are WEB BROWSERS, NOT calculators -- do not confuse
  "abaco" with "abacus". Never guess what a command does from its name: whatis NAME or man NAME.
- cat /dev/user, /dev/sysname, /dev/time, /dev/cputype.  date; cal; seq A B; ps; who.
- test -f FILE / -d DIR.  mkdir cp mv rm; chmod +x FILE.
- rc prompt: $prompt is a STATIC string (rc does NOT re-evaluate it, so command substitution in it
  runs once, not per prompt). For a LIVE prompt, define fn prompt (rc runs it before each prompt):
  fn prompt { prompt=(`{date | awk '{print $4}'}^'% ' ' ') }. Per-user shell startup goes in
  $home/lib/profile (sourced by rcmain for login shells) -- NOT the shared /rc/lib/rcmain.

Networking / software:
- hget URL fetches over http/https to stdout. hget -o FILE URL saves to a file.
  hget -p BODY URL does a POST; hget -r 'Header: value' adds a request header.
- No apt/pkg/brew -- but you are NOT limited to /bin. To get software that isn't
  installed, hget its SOURCE and build it. Source is in /sys/src, or fetch it over
  http from the plan9front mirror, e.g. a single command's source:
      hget -o /tmp/x.c 'http://git.9front.org/plan9front/plan9front/HEAD/sys/src/cmd/x.c/raw'
  then build: 6c/6l for one file, or mk in the source dir for multi-file programs.
  So: check /bin first; if it's missing, fetch the source with hget and build it.

Writing multi-line files: use an rc here-doc with quoted 'EOF' to keep body literal:
    cat > /tmp/f << 'EOF'
    ...literal lines...
    EOF

man NAME (or man N NAME) for details -- sample with sed Nq; never read a whole page.
